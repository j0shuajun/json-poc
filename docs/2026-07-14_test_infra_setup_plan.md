# Step 0. 테스트 실행 인프라 준비 (Plan)

Written at (KST): 2026-07-14 11:20

## 1. 작업 목적

이 프로젝트를 TDD로 진행하려면 먼저 테스트를 실행할 수 있는 환경이 있어야 한다.
현재 저장소에는 `package.json`도, 테스트 실행 설정도 전혀 없다. 이후 모든 단계
(`json-poc`, `quicksort-poc`, `storage`, `contactStore`, `cli`)가 이 인프라 위에서
진행되므로 가장 먼저 준비한다.

## 2. 현재 상태

- 프로젝트에 `package.json`이 없어 `npm test` 같은 표준 명령을 실행할 수 없다.
- 별도 테스트 프레임워크(Jest 등)도 설치되어 있지 않다.

## 3. 목표 상태

- `npm test` 한 줄로 저장소의 모든 `*.test.js` 파일을 실행할 수 있다.
- 외부 패키지 설치 없이 Node.js 표준 내장 테스트 러너(`node:test`)만으로 동작한다
  (JSON PoC 범위를 표준 라이브러리로 한정한 것과 같은 원칙).
- ES Module(`import`/`export`) 문법을 별도 트랜스파일 없이 바로 사용할 수 있다.

## 4. 대표 시나리오

- 개발자가 저장소를 새로 clone한 뒤 별도 설치 없이 `npm test`만 실행하면 테스트 결과를 볼 수 있다.
- `poc/json-poc.test.js` 같은 파일을 추가하면 별도 설정 변경 없이 `npm test`가 이를 찾아 실행한다.

## 5. 변경 접근 방식

- `package.json`을 생성하고 `"type": "module"`로 ES Module을 기본으로 설정한다.
- `"test"` 스크립트를 `node --test`로 등록해 저장소 내 `*.test.js` 파일을 자동 탐색하게 한다.
- `.gitignore`에 `node_modules/`를 추가한다 (현재는 의존성이 없어 비어있지만, 향후 대비).

## 6. 예상 설명 구조

- 별도 다이어그램은 필요하지 않다. 변경 파일과 실행 방법만 result 문서에 정리한다.

## 7. 가정 및 리스크

- 가정: Jest 등 외부 테스트 프레임워크 대신 Node 내장 `node:test`를 사용한다
  (Node 26 기준 안정 기능이며, 별도 설치 없이 표준 라이브러리 원칙을 일관되게 유지할 수 있음).
- 리스크: 낮음. 개발 편의 설정만 추가하며 기존 동작(아직 코드 없음)에 영향이 없다.

## 8. 검증 방법

- `npm test` 실행 시 "명령 없음" 오류 없이 정상적으로 동작(테스트가 아직 없으므로
  "테스트 파일 없음" 상태)하는 것을 확인한다.
