# Step 2. poc/quicksort-poc.js — Quick Sort PoC (Result)

Written at (KST): 2026-07-14 12:10

## 1. 한눈에 보는 요약

- `sortArray(array)` 하나로 숫자 배열을 오름차순 정렬한 새 배열을 얻을 수 있다.
- 원본 배열은 변경되지 않는다.
- 빈 배열/단일 원소/중복값/역순 배열 등 경계 케이스에서도 올바르게 동작함을 테스트로 확인했다.

## 2. 왜 필요했나

Quick Sort를 모른다고 가정하고, 분할정복 알고리즘이 실제로 배열을 어떻게 정렬하는지
직접 구현하며 이해하기 위한 과제 1단계 실습이다.

## 3. 이전 vs 이후

- 이전: 저장소에 정렬 알고리즘이 전혀 없었다.
- 이후: `sortArray([5, 3, 8, 1, 9, 2])`처럼 호출하면 `[1, 2, 3, 5, 8, 9]`를 돌려받는다.

## 4. 대표 시나리오

```js
sortArray([5, 3, 8, 1, 9, 2]);
// => [1, 2, 3, 5, 8, 9]

sortArray([3, 3, 1, 1, 2]); // 중복값 포함
// => [1, 1, 2, 3, 3]

const original = [5, 3, 8, 1];
sortArray(original);
// original은 여전히 [5, 3, 8, 1] (원본 불변)
```

## 5. 독자 입장에서 달라진 점

- `poc/quicksort-poc.js` (구현), `poc/quicksort-poc.test.js` (테스트)가 새로 생겼다.
- 내부적으로 Lomuto partition(마지막 원소를 pivot으로 삼는 방식)을 사용한다:
  pivot보다 작거나 같은 값을 왼쪽으로 모은 뒤, 그 경계에 pivot을 놓고
  왼쪽/오른쪽 구간을 재귀적으로 반복한다.

## 6. 영향 범위

- 새로 추가된 독립 유틸리티라 기존 코드에 영향이 없다. CRUD 앱 요구사항에서
  직접 쓰이지는 않으며, 알고리즘 학습 목적의 별도 산출물이다.

## 7. 제약/후속 작업/한계

- 숫자 배열 정렬만 지원한다 (문자열/객체 정렬은 이번 PoC 범위 밖).
- 안정 정렬(stable sort) 여부는 보장하지 않는다.
