# Step 5. app/cli.js, index.js — 연락처 CRUD 콘솔 UI (Result)

Written at (KST): 2026-07-14 13:15

## 1. 한눈에 보는 요약

- `npm start`로 실행하는 연락처 CRUD 콘솔 앱이 완성됐다.
- 번호 메뉴(목록/ID 검색/키워드 검색/추가/수정/삭제/종료)로 모든 요구 기능을 수행한다.
- 존재하지 않는 ID를 다루거나 잘못된 메뉴 번호를 입력해도 프로그램이 멈추지 않는다.
- 삭제는 실행 전 대상 내용을 보여주고 y/n 확인을 받는 "안전한 삭제"로 동작한다.

## 2. 왜 필요했나

지금까지 만든 `storage.js`(파일 저장)와 `contactStore.js`(생성/조회/수정/삭제 로직)를
사람이 실제로 조작할 수 있는 화면으로 연결해야, 과제 2단계의 "CRUD 콘솔 애플리케이션"이
완성된다.

## 3. 이전 vs 이후

- 이전: 로직은 있었지만 사용자가 입력하고 결과를 볼 수 있는 화면이 없었다.
- 이후: `npm start` 한 번으로 메뉴가 나타나고, 번호만 입력하면 연락처를 관리할 수 있다.

## 5. 독자 입장에서 달라진 점

- `app/cli.js` (메뉴/입출력 로직), `index.js` (실제 진입점), `app/cli.test.js`가 새로 생겼다.
- `app/cli.js`는 `rl.question(prompt)`와 `log(message)`를 인자로 주입받는 구조라서,
  실제 터미널 없이도 정해진 입력 순서로 전체 메뉴 흐름을 자동 테스트할 수 있다
  (`../car`의 `cli.py`가 `input`/`print`를 monkeypatch한 것과 같은 목적).
- `package.json`에 `npm start` 스크립트가 추가됐다.

## 6. 영향 범위

- 사용자가 직접 실행하는 최종 진입점이 새로 생겼다. 이전 단계(`storage`, `contactStore`)의
  동작에는 변화가 없다.

## 7. 제약/후속 작업/한계

- 전화번호/이메일 형식 검증은 하지 않는다(문자열 그대로 저장). 형식 검증이 필요해지면
  `contactStore.createContact`/`updateContact`에 추가하면 된다.
- 동시에 여러 사용자가 같은 파일을 수정하는 상황은 고려하지 않는다(단일 사용자 콘솔 앱).

## 8. Lessons

- Node.js `readline/promises`의 `question()`을 자동화 테스트에서 검증할 때는, 실제
  코드(`app/cli.js`)에 `rl`/`log`를 인자로 주입할 수 있게 설계해 가짜 객체로 테스트했다.
  반면 "실제 터미널처럼 동작하는지"의 수동 검증에서는, `printf '입력들' | node index.js`처럼
  입력을 한 번에 파이프로 흘려보내면 Node의 readline이 EOF를 먼저 감지해 이후 질문이
  응답을 받지 못하고 멈추는 것을 확인했다(대화형 세션과 일괄 파이프 입력은 타이밍이 다름).
  이후에는 자식 프로세스의 stdout 프롬프트에 맞춰 표준입력에 한 줄씩 지연을 두고 써주는
  방식으로 실제 대화형 흐름을 재현해 검증했다.
