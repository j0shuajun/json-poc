# Step 2. poc/quicksort-poc.js — Quick Sort PoC (Plan)

Written at (KST): 2026-07-14 12:05

## 1. 작업 목적

Quick Sort 알고리즘을 "모른다고 가정"하고, 분할정복(divide and conquer)이 실제로
어떻게 배열을 정렬하는지 직접 구현하며 이해한다. 이 PoC는 CRUD 앱 요구사항에서
직접 쓰이지는 않지만, 과제 1단계의 필수 실습 항목이다.

## 2. 현재 상태 (리더 관점)

- 저장소에 정렬 알고리즘이 전혀 없다.
- "pivot을 어떻게 고르는지", "partition이 배열을 어떻게 두 그룹으로 나누는지"에 대한
  코드 수준의 근거가 없다.

## 3. 목표 상태

- `sortArray(array)` 함수 하나로 숫자 배열을 오름차순 정렬한 새 배열을 얻는다.
- 원본 배열은 변경되지 않는다(입력을 그대로 두고 정렬된 복사본을 돌려준다).
- 빈 배열, 원소 1개, 이미 정렬됨, 역순 정렬, 중복값 등 경계 케이스에서도 올바르게 동작한다.

## 4. 대표 시나리오

- `sortArray([5, 3, 8, 1, 9, 2])` → `[1, 2, 3, 5, 8, 9]`
- `sortArray([])` → `[]`
- `sortArray([3, 3, 1, 1, 2])` (중복값 포함) → `[1, 1, 2, 3, 3]`
- 원본 배열 `original`을 `sortArray(original)`에 넘긴 뒤에도 `original` 자체는
  정렬 전 상태 그대로 유지된다.

## 5. 변경 접근 방식

- Lomuto partition 방식을 사용한다: 배열의 마지막 원소를 pivot으로 삼고,
  pivot보다 작거나 같은 값을 왼쪽으로 모은 뒤 pivot을 그 경계에 놓는다.
- `partition(arr, low, high)`이 pivot의 최종 위치(index)를 돌려주면,
  `quicksort(arr, low, high)`가 그 위치를 기준으로 왼쪽/오른쪽 구간을 재귀적으로 정렬한다.
- 두 함수는 배열을 제자리에서(in-place) 정렬하되, 공개 API인 `sortArray`는
  입력을 복사(`[...array]`)한 뒤 그 복사본에 대해서만 in-place 정렬을 수행해
  "원본 불변" 원칙을 지킨다.

## 6. 예상 설명 구조

- 별도 다이어그램 없이, "정렬 전/후 배열 예시" 중심으로 result 문서를 작성한다.

## 7. 가정 및 리스크

- 가정: 정렬 대상은 숫자 배열로 한정한다 (문자열/객체 정렬은 이번 PoC 범위 밖).
- 가정: 안정 정렬(stable sort) 여부는 보장하지 않는다 (Quick Sort의 일반적인 특성을
  그대로 따름).
- 리스크: 낮음. 새로 추가되는 독립적인 유틸리티 함수이며 기존 코드에 영향이 없다.

## 8. 검증 방법

- TDD로 진행: 대표 배열 정렬, 빈 배열/단일 원소, 중복값, 원본 불변성에 대한 테스트를
  먼저 작성해 실패를 확인한 뒤, 최소 구현으로 통과시킨다.
- `npm test`로 전체 테스트 통과를 확인한다.
