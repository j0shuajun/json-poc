# Step 5. app/cli.js, index.js — 연락처 CRUD 콘솔 UI (Plan)

Written at (KST): 2026-07-14 12:55

## 1. 작업 목적

지금까지 만든 `storage.js`(파일 I/O)와 `contactStore.js`(순수 로직)를 실제
터미널에서 사용할 수 있는 메뉴 기반 콘솔 프로그램으로 연결한다. 이 모듈이 완성되면
과제 2단계(CRUD 콘솔 앱)의 요구사항이 모두 충족된다.

## 2. 현재 상태 (리더 관점)

- 연락처를 만들고 찾고 고치고 지우는 로직(`contactStore.js`)과 파일 저장(`storage.js`)은
  준비됐지만, 사용자가 실제로 입력하고 결과를 보는 화면이 없다.

## 3. 목표 상태

- 프로그램을 실행하면 번호로 된 메뉴(전체 목록/ID 검색/키워드 검색/추가/수정/삭제/종료)가
  나타난다.
- 각 메뉴를 선택하면 필요한 값을 입력받아 즉시 결과를 보여주고, 변경 사항은 즉시
  JSON 파일에 저장된다.
- 존재하지 않는 ID를 수정/삭제하려 하면 프로그램이 멈추지 않고 "찾을 수 없다"는
  안내만 보여준다. 삭제는 실행 전에 대상 내용을 보여주고 확인(y/n)을 받는다.

## 4. 대표 시나리오

- 메뉴 1(전체 목록) 선택 → 연락처가 없으면 "등록된 연락처가 없습니다." 안내.
- 메뉴 4(추가) 선택 → 이름/전화/이메일을 순서대로 입력 → "추가되었습니다: [1] 홍길동 | ..." 출력.
- 메뉴 5(수정) 선택 → ID 입력 → 현재 값 표시 → 각 필드를 새 값으로 입력(빈 입력은 유지) →
  "수정되었습니다: ..." 출력.
- 메뉴 6(삭제) 선택 → ID 입력 → 대상 내용 표시 후 "정말 삭제할까요? (y/n)" 확인 →
  `y`가 아니면 "삭제를 취소했습니다."
- 메뉴 0(종료) 선택 → "프로그램을 종료합니다." 출력 후 루프 종료.

## 5. 변경 접근 방식

- `app/cli.js`는 `main({ rl, filePath, log })` 하나를 내보낸다. `rl`은
  `question(prompt): Promise<string>` 메서드를 가진 객체(실제로는
  `node:readline/promises` 인터페이스, 테스트에서는 정해진 답변을 순서대로
  돌려주는 가짜 객체)이고, `log`는 출력 함수(기본값 `console.log`)다.
  이렇게 I/O를 인자로 주입할 수 있게 만들어, 실제 터미널 없이도 메뉴 흐름을
  자동화된 테스트로 검증할 수 있게 한다(car 프로젝트의 `cli.py`가
  `input`/`print`를 monkeypatch로 갈아끼워 테스트한 것과 같은 목적).
- 매 요청마다 `contacts` 배열을 메모리에 들고 있다가, 추가/수정/삭제가 성공할
  때마다 `saveContacts`로 즉시 파일에 반영한다.
- `index.js`는 실제 `readline/promises` 인터페이스와 `app/contacts.json` 경로를
  준비해 `main`을 호출하는 진짜 진입점 역할만 한다(로직 없음).

## 6. 예상 설명 구조

- 메뉴 번호 → 동작 대응표와, 대표 시나리오의 입출력 예시로 result 문서를 구성한다.

## 7. 가정 및 리스크

- 가정: 수정 메뉴에서 특정 필드를 그대로 두고 싶으면 빈 값을 입력하면 된다
  (필드별로 별도 "건너뛰기" 메뉴를 두지 않음 — 콘솔 앱의 단순성을 위한 선택).
- 가정: 삭제는 되돌릴 수 없으므로 실행 전 y/n 확인을 받는다("안전한 삭제" 요구사항 반영).
- 리스크: 낮음. 이 모듈은 이미 테스트된 `storage.js`/`contactStore.js`를 호출만 하는
  조합(integration) 계층이다.

## 8. 검증 방법

- TDD로 진행: 각 메뉴 흐름(목록/ID 검색/키워드 검색/추가/수정/삭제/잘못된 입력)에 대해
  가짜 입력 큐와 출력 캡처를 사용한 테스트를 먼저 작성해 실패를 확인한 뒤, 최소 구현으로
  통과시킨다.
- `npm test`로 전체 테스트 통과를 확인한 뒤, 실제 터미널에서 `npm start`로 수동 시나리오
  (추가 → 목록 → 검색 → 수정 → 삭제 → 재실행 시 데이터 유지)를 확인한다.
